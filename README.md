# Soft Sensing of Intracellular States for CHO Cell Bioprocessing with Ensemble Kalman Filters

This repository accompanies the manuscript **"Soft Sensing of Intracellular States for CHO Cell Bioprocessing with Ensemble Kalman Filters"**, which investigates the inference of unmeasured intracellular states in CHO cell cultures using a Bayesian state estimation framework.

Direct measurement of intracellular nucleotide sugar donors (NSDs) is analytically challenging and incompatible with routine process monitoring. In this work, an Ensemble Kalman Filter (EnKF) is employed to dynamically infer NSD concentrations from standard extracellular measurements, enabling earlier and quality-relevant insight into glycosylation-critical intracellular dynamics.

The repository provides:
- A fully reproducible Jupyter notebook implementing the EnKF framework
- Supporting utilities for simulation, data handling, and result storage
- Scripts to reproduce all figures and analyses reported in the manuscript

The code is organized to ensure portability and reproducibility. All file paths are defined relative to the repository root, and no absolute or machine-specific paths are used.

## Installation

### Prerequisites
- Python 3.14 or higher
- Poetry (for dependency management)

### Setup
1. Clone the repository:
   ```bash
   git clone https://github.com/ly1815/EnKF-Soft-Sensing.git
   cd EnKF-Soft-Sensing
   ```

2. Install dependencies using Poetry:
   ```bash
   poetry install
   ```

3. Activate the virtual environment:
   ```bash
   poetry shell
   ```

## Usage

1. Launch Jupyter Notebook:
   ```bash
   jupyter notebook
   ```

2. Open `JPC_NSD_softsensing.ipynb` and run the cells sequentially to reproduce the analysis.

The notebook includes sections for:
- Package imports and utilities
- Model definition and simulation
- EnKF implementation
- Data analysis and visualization
- Result generation

## Data

- **Raw data**: Located in `data/raw/`, containing experimental measurements from CHO cell cultures (P1.xlsx to P4.xlsx).
- **Results**: Located in `results/`, including generated plots and analysis outputs.
- Note: Large processed data files (e.g., `results_all.pkl`) are not included in the repository due to size limits. They can be regenerated by running the notebook.
